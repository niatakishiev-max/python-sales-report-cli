# Тестирование Sales Report CLI

## Цель тестов

Автотесты проверяют бизнес-логику приложения и функции валидации пользовательского ввода.

Проект специально разделяет:

- бизнес-логику, которую можно тестировать напрямую;
- консольный интерфейс, который отвечает за `input()` и `print()`.

## Как запустить тесты

Установить зависимости:

```bash
python -m pip install -r requirements.txt
```

Запустить тесты:

```bash
python -m pytest
```

## Что покрыто тестами

### Расчёт выручки одной продажи

Проверяется функция `calculate_revenue()`.

Сценарии:

- целая цена и целое количество;
- разные значения цены и количества;
- дробная цена.

### Расчёт общей выручки

Проверяется функция `calculate_total_revenue()`.

Сценарии:

- несколько продаж;
- пустой список продаж.

### Расчёт общего количества товаров

Проверяется функция `calculate_total_quantity()`.

Сценарии:

- несколько продаж;
- пустой список продаж.

### Отчёт по товарам

Проверяется функция `build_product_report()`.

Сценарии:

- группировка продаж по названию товара;
- суммирование количества по каждому товару;
- суммирование выручки по каждому товару;
- пустой список продаж.

### Проверка ввода цены

Проверяется функция `get_float()`.

Сценарии:

- корректное дробное число;
- текст вместо числа;
- ноль;
- отрицательное число.

### Проверка ввода количества

Проверяется функция `get_int()`.

Сценарии:

- корректное целое число;
- текст вместо числа;
- дробное число вместо целого;
- ноль;
- отрицательное число.

## Используемые возможности pytest

- `assert` для проверки результата;
- `pytest.mark.parametrize` для проверки одной логики на разных данных;
- `monkeypatch` для подмены пользовательского ввода;
- `capsys` для проверки текста, который программа выводит через `print()`.
